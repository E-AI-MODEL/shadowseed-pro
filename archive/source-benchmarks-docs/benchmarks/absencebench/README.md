# AbsenceBench

AbsenceBench is de eerste benchmarklane voor deze repo omdat hij het dichtst aansluit op SSL's kernvraag:

> Welke kleine structurele afwezigheid had hier moeten staan, maar staat er niet?

## Focus

- SSL-onderdeel: `Detection-Pass`
- executionstatus: `benchmarkvoorbereiding`
- repo-status runnerroute: `te verifiëren`

## Waarom AbsenceBench eerst

- het benchmarkdoel ligt dicht bij afwezigheidsdetectie
- de baseline versus SSL-conditie is conceptueel goed af te bakenen
- de benchmark dwingt expliciet onderscheid af tussen gewone beantwoording en gerichte detection-pass logica

## Wat deze map nu bevat

- `runkaart.md`: operationele voorbereiding voor de eerste reproduceerbare runroute

## Wat deze map nog niet claimt

- geen hard geverifieerde actuele runner
- geen bevestigd definitief startcommando
- geen voltooide benchmarkscore
